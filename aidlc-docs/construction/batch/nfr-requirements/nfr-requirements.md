# Batch Unit - NFR Requirements

## NFR-B01: 성능
- 100건 미만 문서 처리 기준 전체 임베딩 수 분 이내 완료
- 증분 임베딩은 변경 감지 포함 수 초~수십 초 이내
- OpenAI API 호출이 병목 → API rate limit에 따라 조절

## NFR-B02: 안정성
- 개별 문서 실패 시 전체 프로세스 중단하지 않음 (fail-safe)
- 실패 문서는 error 상태로 기록, 다음 실행 시 재시도 가능
- 예외 발생 시 적절한 에러 메시지 콘솔 출력

## NFR-B03: 데이터 무결성
- 임베딩과 메타데이터 업데이트는 문서 단위로 원자적 처리
- 중간 실패 시 부분 처리된 문서는 다음 실행에서 재처리 (해시 불일치로 감지)
- 삭제 처리 시 ChromaDB, KùzuDB, SQLite 모두 정리

## NFR-B04: 운영성
- 실행 결과를 콘솔에 명확히 출력 (처리/건너뛰기/실패/삭제 건수)
- SQLite에 실행 이력 저장 (추후 모니터링 확장 가능)
- .bat 파일로 Windows 작업 스케줄러 등록 가능

## NFR-B05: 유지보수성
- 문서 파서는 파일 형식별로 분리하여 추후 PDF 등 추가 용이
- 설정값(청크 크기, API URL 등)은 config.py에서 중앙 관리
- 로깅은 Python logging 모듈 사용

## NFR-B06: 오프라인 호환성
- 모든 Python 패키지는 wheel/tar.gz 오프라인 설치 가능해야 함
- 외부 네트워크 의존 없음 (내부 OpenAI API만 사용)
