# Batch Unit - Business Logic Model

## 1. 메인 실행 흐름

```
batch.py 실행 (--full 옵션 여부)
    |
    v
[모드 결정] full=True → 전체 재임베딩 / full=False → 증분 임베딩
    |
    v
[변경 감지] (증분 모드만)
    |-- 문서 디렉토리 스캔
    |-- SQLite에서 기존 해시 조회
    |-- 파일별 SHA-256 해시 비교
    |-- ChangeSet 생성 (new/modified/deleted/unchanged)
    |
    v
[전체 모드: 기존 데이터 삭제]
    |-- ChromaDB 컬렉션 초기화
    |-- KùzuDB 그래프 데이터 삭제
    |-- SQLite 문서 메타데이터 초기화
    |
    v
[대상 문서 처리] (new + modified 파일 대상)
    |
    +-- [문서 파싱]
    |     |-- 파일 형식 판별 (.txt/.csv/.md)
    |     |-- 형식별 파서로 텍스트 추출
    |     |-- 지원하지 않는 형식 → 건너뛰기 + 경고
    |
    +-- [청킹]
    |     |-- 텍스트를 적절한 크기로 분할
    |     |-- 청크별 메타데이터 부여 (doc_id, index, 파일명)
    |
    +-- [임베딩]
    |     |-- 내부 OpenAI API로 청크 임베딩
    |     |-- ChromaDB에 벡터 저장 (upsert)
    |
    +-- [엔티티 추출]
    |     |-- LLM으로 청크에서 엔티티/관계 추출
    |     |-- KùzuDB에 엔티티/관계 저장 (upsert)
    |
    +-- [메타데이터 업데이트]
          |-- SQLite에 문서 해시, 상태, 청크 수 저장
    |
    v
[삭제 문서 처리] (deleted 파일 대상)
    |-- ChromaDB에서 해당 문서 벡터 삭제
    |-- KùzuDB에서 해당 문서 엔티티/관계 삭제
    |-- SQLite에서 메타데이터 삭제
    |
    v
[결과 출력]
    |-- 처리 건수 (total/processed/skipped/failed/deleted)
    |-- 실행 시간
    |-- SQLite에 BatchResult 저장
```

## 2. 문서 파싱 로직

| 파일 형식 | 파싱 방법 |
|---|---|
| .txt | 전체 텍스트 읽기 (UTF-8) |
| .csv | 행 단위 읽기, 헤더 + 데이터 행을 텍스트로 변환 |
| .md | 전체 텍스트 읽기 (마크다운 그대로) |
| 기타 | 건너뛰기 + "복호화 후 .txt/.csv로 변환해주세요" 경고 출력 |

## 3. 청킹 전략

- **방식**: 고정 크기 + 오버랩
- **청크 크기**: ~500 토큰 (약 1500자)
- **오버랩**: ~50 토큰 (약 150자)
- **CSV 특수 처리**: 헤더를 각 청크에 포함하여 컨텍스트 유지
- **구분자 우선**: 문단/줄바꿈 경계에서 분할 우선 시도

## 4. 엔티티 추출 전략

- **방식**: LLM 프롬프트 기반 추출
- **추출 대상**: 개념, 용어, 인물, 조직, 시스템, 프로세스
- **관계 유형**: RELATED_TO, PART_OF, USES, DEPENDS_ON, MANAGES
- **프롬프트**: 청크 텍스트를 입력으로 JSON 형태의 엔티티/관계 목록 추출
- **중복 처리**: 동일 엔티티명은 KùzuDB에서 MERGE로 중복 방지
