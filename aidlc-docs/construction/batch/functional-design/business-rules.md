# Batch Unit - Business Rules

## BR-01: 지원 파일 형식
- .txt, .csv, .md 파일만 처리
- 그 외 확장자는 건너뛰고 경고 메시지 출력: "지원하지 않는 형식입니다. 복호화 후 .txt/.csv로 변환하여 다시 배치해주세요."

## BR-02: 변경 감지 (증분 모드)
- 파일의 SHA-256 해시를 계산하여 SQLite에 저장된 이전 해시와 비교
- 해시가 다르면 "변경됨"으로 판정 → 재임베딩 대상
- SQLite에 없는 파일은 "신규"로 판정 → 임베딩 대상
- SQLite에 있지만 디렉토리에 없는 파일은 "삭제됨"으로 판정 → 삭제 처리

## BR-03: 전체 재임베딩 모드
- `--full` 옵션 시 기존 데이터 전체 삭제 후 모든 문서 재처리
- ChromaDB 컬렉션 초기화, KùzuDB 그래프 삭제, SQLite 메타데이터 초기화

## BR-04: 청킹 규칙
- 청크 크기: ~500 토큰 (약 1500자)
- 오버랩: ~50 토큰 (약 150자)
- 문단/줄바꿈 경계에서 분할 우선
- CSV: 헤더를 각 청크에 포함

## BR-05: 임베딩 실패 처리
- 개별 문서 임베딩 실패 시 해당 문서만 건너뛰고 계속 진행
- 실패한 문서는 status="error"로 기록
- 전체 실행은 중단하지 않음

## BR-06: 삭제 문서 정리
- 삭제된 문서의 벡터(ChromaDB), 엔티티/관계(KùzuDB), 메타데이터(SQLite) 모두 제거
- 부분 삭제 실패 시 로그 출력 후 계속 진행

## BR-07: 엔티티 추출 규칙
- LLM에 JSON 형태로 엔티티/관계 추출 요청
- 추출 실패 시 해당 청크의 엔티티 추출만 건너뛰고 벡터 임베딩은 유지
- 동일 엔티티명은 KùzuDB MERGE로 중복 방지

## BR-08: 실행 결과 출력
- 콘솔에 출력: 전체 파일 수, 처리 수, 건너뛴 수, 실패 수, 삭제 수, 실행 시간
- SQLite에 BatchResult 저장 (실행 이력)

## BR-09: 문서 디렉토리
- 기본 경로: 프로젝트 루트의 `documents/` 디렉토리
- config.py에서 경로 변경 가능
