# Frontend Unit - Functional Design

## UI 컴포넌트 구조

```
+------------------------------------------+
|  NH-RAG 채팅                    [새 대화] |
+------------------------------------------+
|                                          |
|  [채팅 메시지 영역]                       |
|                                          |
|  사용자: 질문 텍스트                      |
|                                          |
|  AI: 응답 텍스트 (스트리밍)               |
|      📄 출처:                            |
|      - test.txt: "원문 발췌..."           |
|                                          |
+------------------------------------------+
|  [입력창                    ] [전송 버튼] |
+------------------------------------------+
```

## Business Rules

### BR-F01: 메시지 전송
- 빈 입력은 전송 불가 (버튼 비활성화)
- Enter 키로 전송, Shift+Enter로 줄바꿈
- 전송 후 입력창 초기화
- 스트리밍 중에는 전송 버튼 비활성화

### BR-F02: SSE 스트리밍 수신
- `POST /api/chat` 호출 후 SSE 이벤트 수신
- `token` 이벤트: 실시간으로 AI 메시지 영역에 텍스트 추가
- `sources` 이벤트: 출처 정보를 AI 메시지 하단에 렌더링
- `done` 이벤트: 스트리밍 완료, 전송 버튼 재활성화
- `error` 이벤트: 에러 메시지 표시
- `session` 이벤트: session_id 저장

### BR-F03: 세션 관리
- session_id를 메모리(변수)에 저장
- 매 요청 시 session_id 포함하여 전송
- 브라우저 닫으면 session_id 소멸 (세션 기반)

### BR-F04: 새 대화
- "새 대화" 버튼 클릭 시 채팅 영역 초기화
- `POST /api/chat/new` 호출하여 새 session_id 발급
- 이전 대화 내용 화면에서 제거

### BR-F05: 출처 표시
- 출처가 있을 때만 표시
- 문서명 + 원문 발췌 (200자 제한)
- 접이식(collapsible) 형태로 표시

### BR-F06: 자동 스크롤
- 새 메시지/토큰 추가 시 채팅 영역 하단으로 자동 스크롤

### BR-F07: 마크다운 렌더링
- AI 응답에 마크다운이 포함될 수 있으므로 기본적인 마크다운 렌더링 지원
- 코드 블록, 볼드, 리스트 등 기본 문법
