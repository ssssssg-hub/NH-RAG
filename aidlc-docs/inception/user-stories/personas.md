# NH-RAG 사용자 페르소나

## Persona 1: 일반 사용자 (End User)

| 항목 | 내용 |
|---|---|
| **이름** | 김사원 |
| **역할** | 내부 직원 (기술직/사무직 혼합) |
| **기술 수준** | 중급 - 웹 브라우저 사용에 익숙, LLM 채팅 경험 있음 |
| **주요 목표** | 사내 지식 문서 및 프로그램 관련 정보를 빠르게 검색하여 업무에 활용 |
| **Pain Point** | 키워드 검색으로는 원하는 정보를 찾기 어려움, 문서가 분산되어 있음 |
| **사용 빈도** | 일 수회 |
| **기대** | 자연어로 질문하면 정확한 답변과 출처를 받을 수 있음 |

## Persona 2: 관리자 (Admin)

| 항목 | 내용 |
|---|---|
| **이름** | 이관리 |
| **역할** | 시스템 관리자 (= 일반 사용자와 동일인물 가능) |
| **기술 수준** | 상급 - Python 실행, 스케줄러 등록, 파일 관리 가능 |
| **주요 목표** | 문서 임베딩 관리, 시스템 운영, 문서 최신 상태 유지 |
| **Pain Point** | 문서 업데이트 시 수동으로 임베딩을 갱신해야 함 |
| **사용 빈도** | 주 1~2회 (문서 업데이트 주기) |
| **기대** | 간단한 명령으로 문서 임베딩을 실행하고 상태를 확인할 수 있음 |

## 페르소나-스토리 매핑 요약

| 페르소나 | 관련 Feature 그룹 |
|---|---|
| 김사원 (일반 사용자) | 채팅, RAG 검색, 대화 이력 |
| 이관리 (관리자) | Batch 임베딩, 문서 관리, 채팅(겸용) |
