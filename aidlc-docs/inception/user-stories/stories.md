# NH-RAG 사용자 스토리

## 우선순위 범례
- **Must**: 1차 개발에 반드시 포함
- **Should**: 1차 개발에 포함 권장
- **Could**: 여유 시 포함

---

## Feature Group 1: 채팅 인터페이스 (FR-01)

### US-01: 질문 입력 및 전송 [Must]
**As a** 일반 사용자 (김사원)
**I want to** 웹 채팅 화면에서 자연어로 질문을 입력하고 전송할 수 있다
**So that** 사내 문서에서 필요한 정보를 빠르게 찾을 수 있다

**Acceptance Criteria:**
- Given 사용자가 채팅 페이지에 접속했을 때, When 페이지가 로드되면, Then 텍스트 입력창과 전송 버튼이 표시된다
- Given 사용자가 질문을 입력했을 때, When 전송 버튼을 클릭하거나 Enter를 누르면, Then 질문이 서버로 전송되고 채팅 영역에 사용자 메시지가 표시된다
- Given 빈 입력 상태일 때, When 전송을 시도하면, Then 전송되지 않는다

**의존성**: 없음 (기반 기능)

---

### US-02: 스트리밍 응답 표시 [Must]
**As a** 일반 사용자 (김사원)
**I want to** LLM 응답이 실시간으로 글자 단위로 나타나는 것을 볼 수 있다
**So that** 응답 대기 시간 동안 진행 상황을 확인할 수 있다

**Acceptance Criteria:**
- Given 질문이 전송되었을 때, When 서버가 응답을 생성하면, Then 응답 텍스트가 실시간으로 한 글자씩 채팅 영역에 표시된다
- Given 스트리밍 중일 때, When 응답이 완료되면, Then 전체 응답이 채팅 영역에 완성된 형태로 표시된다
- Given LLM API 호출이 실패했을 때, When 에러가 발생하면, Then 사용자에게 "응답 생성에 실패했습니다. 다시 시도해주세요." 메시지가 표시된다

**의존성**: US-01

---

## Feature Group 2: RAG 검색 (FR-03)

### US-03: 하이브리드 검색 기반 답변 [Must]
**As a** 일반 사용자 (김사원)
**I want to** 내 질문에 대해 Vector 검색과 Graph 검색이 결합된 정확한 답변을 받을 수 있다
**So that** 키워드 검색보다 더 정확하고 맥락에 맞는 정보를 얻을 수 있다

**Acceptance Criteria:**
- Given 사용자가 질문을 전송했을 때, When 백엔드가 검색을 수행하면, Then Vector(ChromaDB) 검색과 Graph(KùzuDB) 검색이 동시에 실행되고 결합된 결과가 LLM context로 전달된다
- Given 검색 결과가 있을 때, When LLM이 응답을 생성하면, Then 검색된 문서 내용을 기반으로 한 답변이 제공된다
- Given 관련 문서가 없을 때, When 검색 결과가 비어있으면, Then "관련 문서를 찾을 수 없습니다" 메시지와 함께 일반 LLM 응답을 제공한다

**의존성**: US-01, Batch 임베딩(US-07) 완료 필요

---

### US-04: 출처 표시 [Must]
**As a** 일반 사용자 (김사원)
**I want to** 답변에 참조된 문서명과 관련 원문 발췌를 확인할 수 있다
**So that** 답변의 신뢰성을 검증하고 원본 문서를 참고할 수 있다

**Acceptance Criteria:**
- Given LLM이 문서 기반 답변을 생성했을 때, When 응답이 표시되면, Then 답변 하단에 참조 문서명과 관련 원문 발췌가 표시된다
- Given 여러 문서가 참조되었을 때, When 출처가 표시되면, Then 각 문서별로 구분되어 표시된다
- Given 검색 결과 없이 일반 응답인 경우, When 응답이 표시되면, Then 출처 영역이 표시되지 않는다

**의존성**: US-03

---

## Feature Group 3: 대화 이력 (FR-02)

### US-05: 세션 내 대화 이력 유지 [Must]
**As a** 일반 사용자 (김사원)
**I want to** 현재 세션에서 이전 대화 내용을 스크롤하여 확인할 수 있다
**So that** 이전 질문과 답변을 참고하며 후속 질문을 할 수 있다

**Acceptance Criteria:**
- Given 사용자가 여러 질문을 했을 때, When 채팅 영역을 스크롤하면, Then 모든 이전 질문과 답변이 순서대로 표시된다
- Given 이전 대화가 있을 때, When 새 질문을 전송하면, Then 이전 대화 맥락이 LLM에 전달되어 멀티턴 대화가 가능하다
- Given 브라우저를 닫았을 때, When 다시 접속하면, Then 이전 대화 이력은 초기화된다

**의존성**: US-01, US-02

---

### US-06: 새 대화 시작 [Should]
**As a** 일반 사용자 (김사원)
**I want to** 현재 대화를 초기화하고 새 대화를 시작할 수 있다
**So that** 새로운 주제로 깨끗한 상태에서 질문할 수 있다

**Acceptance Criteria:**
- Given 대화가 진행 중일 때, When "새 대화" 버튼을 클릭하면, Then 채팅 영역이 초기화되고 새 세션이 시작된다
- Given 새 대화가 시작되었을 때, When 질문을 전송하면, Then 이전 대화 맥락 없이 새로운 대화로 처리된다

**의존성**: US-05

---

## Feature Group 4: Batch 임베딩 (FR-04)

### US-07: 수동 문서 임베딩 실행 [Must]
**As a** 관리자 (이관리)
**I want to** 명령어 또는 .bat 파일로 문서 임베딩을 실행할 수 있다
**So that** 새로운 문서나 변경된 문서를 검색 가능한 상태로 만들 수 있다

**Acceptance Criteria:**
- Given 지정 디렉토리에 문서(.txt, .csv, .md)가 있을 때, When 임베딩 스크립트를 실행하면, Then 문서가 Vector(ChromaDB)와 Graph(KùzuDB)에 임베딩된다
- Given 임베딩이 완료되었을 때, When 실행 결과를 확인하면, Then 처리된 문서 수, 성공/실패 건수가 콘솔에 출력된다
- Given 지원하지 않는 파일 형식이 있을 때, When 임베딩을 실행하면, Then 해당 파일은 건너뛰고 경고 메시지가 출력된다

**의존성**: 없음 (기반 기능)

---

### US-08: 증분 임베딩 [Must]
**As a** 관리자 (이관리)
**I want to** 변경된 문서만 자동으로 감지하여 재임베딩할 수 있다
**So that** 전체 재처리 없이 효율적으로 문서를 최신 상태로 유지할 수 있다

**Acceptance Criteria:**
- Given 이전에 임베딩된 문서가 있을 때, When 임베딩을 실행하면, Then 파일 해시 비교를 통해 변경된 문서만 재임베딩된다
- Given 문서가 삭제되었을 때, When 임베딩을 실행하면, Then 해당 문서의 임베딩 데이터가 ChromaDB와 KùzuDB에서 제거된다
- Given 모든 문서가 변경되지 않았을 때, When 임베딩을 실행하면, Then "변경된 문서 없음" 메시지가 출력되고 빠르게 종료된다

**의존성**: US-07

---

### US-09: 스케줄 기반 자동 임베딩 (.bat) [Should]
**As a** 관리자 (이관리)
**I want to** Windows 작업 스케줄러에 등록할 수 있는 .bat 파일이 제공된다
**So that** 주기적으로 자동 임베딩을 실행할 수 있다

**Acceptance Criteria:**
- Given .bat 파일이 제공되었을 때, When Windows 작업 스케줄러에 등록하면, Then 설정된 주기에 따라 임베딩이 자동 실행된다
- Given .bat 파일을 직접 실행했을 때, When 더블클릭하면, Then 수동 임베딩과 동일하게 동작한다

**의존성**: US-07, US-08

---

### US-10: 전체 재임베딩 [Should]
**As a** 관리자 (이관리)
**I want to** 필요 시 모든 문서를 처음부터 다시 임베딩할 수 있다
**So that** 임베딩 데이터에 문제가 있을 때 초기화할 수 있다

**Acceptance Criteria:**
- Given 관리자가 전체 재임베딩을 원할 때, When `--full` 옵션으로 스크립트를 실행하면, Then 기존 임베딩 데이터를 모두 삭제하고 전체 문서를 재임베딩한다
- Given 전체 재임베딩 중일 때, When 처리가 진행되면, Then 진행 상황(n/전체)이 콘솔에 출력된다

**의존성**: US-07

---

## Feature Group 5: 문서 관리 (FR-05)

### US-11: 문서 배치 및 형식 안내 [Must]
**As a** 관리자 (이관리)
**I want to** 지정된 디렉토리에 문서를 배치하면 Batch 서비스가 처리한다
**So that** 별도 업로드 UI 없이 파일 시스템으로 문서를 관리할 수 있다

**Acceptance Criteria:**
- Given 지정 디렉토리가 설정되어 있을 때, When 해당 디렉토리에 .txt/.csv/.md 파일을 배치하면, Then 다음 임베딩 실행 시 해당 문서가 처리된다
- Given 지원하지 않는 형식(.xlsx, .docx 등)의 파일이 있을 때, When 임베딩을 실행하면, Then 해당 파일은 건너뛰고 "복호화 후 .txt/.csv로 변환하여 다시 배치해주세요" 안내 메시지가 출력된다

**의존성**: US-07

---

## 스토리 의존성 맵

```
US-01 (질문 입력) ─────┬──> US-02 (스트리밍) ──> US-05 (대화 이력) ──> US-06 (새 대화)
                       │
                       └──> US-03 (하이브리드 검색) ──> US-04 (출처 표시)
                                  ^
                                  │
US-07 (수동 임베딩) ──┬──> US-08 (증분 임베딩) ──> US-09 (.bat 스케줄)
                      ├──> US-10 (전체 재임베딩)
                      └──> US-11 (문서 배치)
```

| FR-02 대화 이력 | US-05, US-06 |
| FR-03 RAG 검색 | US-03, US-04 |
| FR-04 Batch 임베딩 | US-07, US-08, US-09, US-10 |
