# RAG 서비스 요구사항 정의서

## 1. 프로젝트 개요

### 1.1 서비스 비전

외부 인터넷과의 연결이 불가능한 내부망 환경에서 내부 OpenAI 기반의 LLM API를 활용해 내부 문서기반 검색을 편리하게 할 수 있는 RAG 서비스 구축

### 1.2 핵심 가치 제안

**사용자 관점**

- 사용자가 서비스로 올려둔 웹페이지에서 LLM과 자연스럽게 대화하며 내부 지식 문서를 키워드 검색이 아닌 Vector + Graph 기반의 하이브리드 검색을 통해 지식을 빠르게 얻는다

## 2. 서비스 구성

### 2.1 주요 구성 요소

- **채팅화면 인터페이스**: 내부지식이 있는 백엔드 서비스와의 API통신 혹은 WebSocket을 통한 통신으로 사용자가 질의하고 응답을 받는 구조의(ex: ChatGPT, Claude 등과 같은 LLM 사이트와 유사한 형태) 화면 구성
- **Vector+Graph 기반의 백엔드 서버 시스템**: Vector(ChromaDB) + Graph(KùzuDB) 기반의 임베딩 로직과 임베딩한 값을 프롬프트화 해서 LLM에 서빙하고 받은 응답값을 채팅화면과 인터페이스 시키는 AI 비즈니스로직이 포함된 백엔드 서버 시스템
- **Batch 서비스**: 문서 임베딩을 주기적으로 실행시키기 위한 서비스. 동일하게 Vector + Graph 기반의 임베딩 로직을 활용하며, 문서 임베딩과 사용자 질의 임베딩은 동일한 임베딩 로직을 타게 됨.
- **데이터 저장소**: 채팅 히스토리등 로그화 시킬 내용들이 들어갈 저장소. 꼭 RDBMS가 아니더라도 활용가능하다면 어느 형태이든 가능

## 3. 핵심 기능 요구사항

**요청할 사항임.** 핵심기능을 온전히 정의하지 못했으므로 같이 정의하는 단계를 진행해나가야함.
여기서 가장 중요한 점은 외부와의 연결이 불가능한 상태라는 가정하에 진행될 것이며
Python 버전은 3.11.9를 기반으로 개발에 진행.
JS는 VanillaJS를 사용해 웹 진행.
배포는 local에만 이루어질 예정.
설치가 필요한 그 어떤 항목이라도(ex:MySQL, Elastic Search ...) 사용을 최대한 보수적으로 잡을것. 이유는 내부망에 해당 서비스들을 반입하기가 까다롭기 때문.
